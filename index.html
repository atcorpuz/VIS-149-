What's the difference between a regular array and a 2D array in Java?
A regular array holds a list of elements of the same type, while a 2D array holds multiple arrays â€” essentially a grid. You can think of it as a table where each cell is accessed using two indices: one for the row and one for the column.
// Example: creating and filling a 2D array
int[][] grid = new int[3][4];
// Filling the grid
for (int row = 0; row < 3; row++) {
    for (int col = 0; col < 4; col++) {
        grid[row][col] = row + col;
    }
}
// Accessing a specific cell
System.out.println(grid[1][2]);

What was confusing: When I asked about "jagged arrays," the explanation got too technical too quickly. It assumed I already knew memory allocation terms.

Compared to human teaching: Claude was faster and more patient, but also less emotionally aware. A human might have noticed when I was confused; Claude waited for me to admit it.

What's the main difference between an Array and an ArrayList?
Arrays are fixed in size â€” once you create them, you can't add or remove elements. ArrayLists, on the other hand, can grow or shrink dynamically. They also come with built-in methods like add(), remove(), and contains().

names.add("Helio");

System.out.println(names.contains("Helio"));

What exactly is an "object" in Java?
An object is an instance of a class â€” a concrete thing created from a blueprint. If a class defines what something *is* and what it *can do*, then an object is that thing in action.

  int age;

    this.age = age;
  }

}
